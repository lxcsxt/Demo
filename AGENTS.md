# AGENTS.md

## 工作方式
- 回答与修改默认基于当前仓库上下文进行。
- 当任务涉及代码评审时，先读取 `docs/ai-rules/code-review.md`，并严格按其中规则输出。
- 当任务涉及生成单元测试时，先读取 `docs/ai-rules/unit-test.md`，并严格按其中规则输出。
- 当任务涉及测试覆盖率分析时，先读取 `docs/ai-rules/test-coverage.md`，并严格按其中规则输出。
- 如果一次任务同时涉及评审、生成单测、覆盖率分析，先确认当前任务目标，再只应用对应规则，避免混用输出格式。

## 输出要求
- 代码评审必须给出文件路径和行号。
- 默认不要直接修改代码，除非我明确要求你改。
- 默认优先阅读当前 diff、当前打开文件和直接调用链。
- 结论优先，解释随后给出。

## Java 约定
- Java 项目优先使用 Java 语言风格与 Spring Boot 语境分析。
- 生成测试时优先使用 JUnit 5、Mockito、Spring Boot Test。
